# Definition Of Done

Questa checklist definisce il minimo accettabile per chiudere un dataset o una release. Per i controlli usa anche [VALIDATION_PLAYBOOK.md](./VALIDATION_PLAYBOOK.md).

## RAW

- Fonte ufficiale identificata e citabile
- Modalita di acquisizione ripetibile
- Provenance registrata
- Nessuna modifica manuale non documentata

## CLEAN

- Regole di pulizia esplicite
- Tipi e chiavi controllati
- Anomalie note annotate
- Output ricostruibile dal RAW

## MART

- Ogni metrica risponde a una domanda
- Join e aggregazioni spiegabili
- Colonne finali nominate in modo chiaro
- Tavola finale abbastanza stabile per analisi o dashboard

## DOCS

- README public-first aggiornato
- Metodo e limiti dichiarati
- Stato del dataset esplicito
- Link interni funzionanti

## RELEASE

- Changelog o release notes pronte
- Check di validazione eseguiti
- Public readiness check passato
- Versione coerente con maturita del dataset
