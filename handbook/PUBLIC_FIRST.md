# Public First

La parte alta di ogni README deve essere leggibile da chi non conosce il dataset, il dominio o il Lab.

## Struttura consigliata

1. Perche conta
2. Domande
3. Metriche
4. Ipotesi o limiti
5. Stato

## Regole di linguaggio

- apri con il problema, non con la pipeline;
- evita gergo nella parte alta;
- spiega acronimi e codici prima di usarli;
- separa fatti, ipotesi e limiti.

## Esempio minimo

**Perche conta**  
Questo dataset aiuta a capire come cambia l'accesso a un servizio pubblico nel tempo.

**Domande**  
Quante unita osserviamo? Dove aumentano o diminuiscono? Con quale frequenza si aggiorna il dato?

**Metriche**  
Numero totale, tasso per abitante, variazione anno su anno.

**Ipotesi o limiti**  
La copertura territoriale non e uniforme. Alcuni periodi hanno dati mancanti.

**Stato**  
Release candidata / release pubblica / in revisione.
