# Projects

`projects/` è il registro leggero dei filoni già promossi a repo dedicate.

Qui non vivono dataset, SQL o notebook di lavoro.
Quel materiale deve stare nelle repo progetto.

Qui restano solo schede brevi per tenere leggibile lo stato del Lab.

## Struttura

```text
projects/
  README.md
  <slug>.md
```

## Regola pratica

- `preanalysis/` = candidato ancora dentro `dataciviclab`
- `projects/<slug>.md` = filone già promosso a repo dedicata

Lo stato del progetto vive nella scheda stessa, non nella gerarchia delle cartelle.

Le Discussion pubbliche restano fuori da questa cartella e continuano a vivere nel repo `dataciviclab`.

## Scheda minima

Ogni scheda dovrebbe indicare almeno:

- repo del progetto
- domanda principale o asse del progetto
- stato attuale
- classificazione operativa, se utile
- link alle Discussion pubbliche rilevanti, quando disponibili
- prossimo passo
