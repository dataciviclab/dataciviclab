# Projects

`projects/` e il registro leggero dei filoni gia promossi a repo dedicate.

Qui non vivono dataset, SQL o notebook di lavoro.
Quel materiale deve stare nelle repo progetto.

Qui restano solo schede brevi per tenere leggibile lo stato del Lab.

## Struttura

```text
projects/
  README.md
  active/
    README.md
    <slug>.md
  archive/
    README.md
    <slug>.md
```

## Regola pratica

- `preanalysis/` = candidato ancora dentro `dataciviclab`
- `projects/active/` = progetto promosso a repo dedicata
- `projects/archive/` = progetto chiuso, congelato o lasciato come reference

Le Discussion pubbliche restano fuori da queste cartelle e continuano a vivere nel repo `dataciviclab`.
